Project Overview

Primary Objective

A modular, Streamlit-based data processing pipeline designed for media analysts to clean, transform, enrich, and translate media coverage datasets with minimal technical expertise.

What This Application Does

The RDB Media Analytics Pipeline allows you to upload raw media data (CSV or Excel files) and apply a configurable sequence of processing steps. Each step is an independent module that performs a specific transformation on your data. You can run all steps in sequence or pick only the ones you need.

Available Modules

Cleaning

Header Normalization

Standardizes column names by trimming whitespace and converting to Title Case for consistency across datasets.

Cleaning

Duplicate Remover

Removes duplicate rows based on all columns or specific columns you choose, ensuring each record is unique.

Transformation

Date Metadata Extraction

Auto-detects date columns and adds Year, Month, Day, and Quarter fields for time-based analysis and reporting.

Transformation

Column Translation

Translates text columns to a target language using Google Translate, with smart caching and similarity grouping for efficiency.

Analysis

Topic Clustering

Automatically groups text data into thematic clusters using TF-IDF and K-Means, with rich visualizations for topic discovery.

Merge

CSV Merge

Combines multiple CSV files into one dataset, tagging each row with a Country Code extracted from the source filename.

Key Features

  • Web-based Interface — No command-line knowledge required. Upload, process, and download through your browser.
  • Modular Architecture — Each processing step is independent. New modules can be added without modifying existing code.
  • Multi-format Support — Handles CSV files with various encodings (UTF-8, Latin-1, CP-1252) and delimiters (comma, tab, semicolon), plus Excel files.
  • Real-time Progress — A progress bar tracks each step of the pipeline as it runs.
  • Preview & Download — Preview the first 100 rows of your processed data before downloading as CSV or Excel.

Directory Descriptions

src/cleaning/

Modules that clean and standardize raw data without altering its meaning. These should generally be run first in the pipeline.

  • normalize_headers.py — Trims whitespace and title-cases all column names
  • removing_duplicates.py — Removes duplicate rows (full or column-specific)

src/transformation/

Modules that enrich or transform the data by adding new information or converting existing content.

  • add_dates_metadata.py — Extracts Year, Month, Day, Quarter from date columns
  • translate_columns.py — Translates text columns to a target language

src/utils/

Shared utility functions used by all modules. Not a pipeline step itself.

  • csv_handler.py — Robust CSV/Excel reading and writing with encoding/delimiter auto-detection

src/clustering/

Text analysis modules that discover patterns and group content by theme.

  • topic_clustering.py — TF-IDF vectorization + K-Means clustering with PCA visualization
  • config.py — Default parameters for clustering (n_clusters, TF-IDF settings, etc.)

src/merge/

Modules for combining multiple data files into a single dataset.

  • merge_csv.py — Merges multiple CSV files with Country Code tagging from filenames

I-P-O Model (Input – Process – Output)

Application-Level I-P-O

INPUT

What you provide:

  • A CSV or Excel file containing media data
  • Pipeline step selections (which modules to run)
  • Optional parameters (columns for dedup, target language, etc.)

Requirements:

  • Python 3.x installed
  • Dependencies from requirements.txt
  • Internet connection (for translation)

PROCESS

What happens:

  1. File is uploaded via Streamlit
  2. CSV Handler reads the file (auto-detects encoding/delimiter)
  3. Selected pipeline steps execute in order
  4. Each step reads → transforms → writes back to temp file
  5. Progress bar updates after each step

OUTPUT

What you receive:

  • Preview table (first 100 rows) in the browser
  • Downloadable CSV file
  • Downloadable Excel (.xlsx) file
  • Cleaned, enriched, and/or translated data

Full Application Workflow

This diagram shows the complete pipeline from file upload to final download. Each processing step is optional and can be toggled independently.

Pipeline Execution Details

When you click "Run Pipeline", the application executes the following sequence:

  1. The uploaded file is saved to a temporary location.
  2. For each selected step (in order), the system:
    • Reads the current temp file using csv_handler.read_csv()
    • Applies the module's transformation function
    • Writes the result back to the temp file using csv_handler.write_csv()
    • Updates the progress bar
  3. After all steps complete, the final DataFrame is displayed as a preview table.
  4. Download buttons are generated for CSV and Excel formats.
Tip

Steps that are not selected are simply skipped. The data flows directly from one active step to the next. You can run just one step if that's all you need.

Header Normalization

When to Use

Duplicate Remover

Cleaning Module

src/cleaning/removing_duplicates.py

Purpose

Identifies and removes duplicate rows from your dataset. You can check for duplicates across all columns (exact match) or specify particular columns to compare (partial match). This is essential when consolidating media data from overlapping sources.

When to Use

  • Merging data from multiple media monitoring platforms
  • The same article appears multiple times with slightly different metadata
  • You want each media mention to be counted only once in your analysis

Workflow Diagram

Parameters

ParameterTypeDefaultDescription

Error Handling

  • Invalid columns — If you specify column names that don't exist, a ValueError is raised listing the unrecognized names.
  • Case mismatch — Column matching is case-insensitive, so "article title" matches "Article Title".
Tip for Media Analysts

For media data, de-duplicating on "Title" + "Source" is usually more useful than all columns, since timestamps and minor metadata often differ between duplicate entries.

Date Metadata Extraction

Transformation Module

src/transformation/add_dates_metadata.py

Purpose

Automatically detects date columns in your dataset and generates four new derived columns — Year, Month, Day, and Quarter — inserted immediately after the detected date column. These are essential for time-based filtering, trend analysis, and quarterly reporting.

Date Column Auto-Detection

The module searches for a date column in this priority order:

  1. date
  2. date_published
  3. published_date
  4. published
  5. datetime
  6. timestamp
  7. Any column name containing "date"

Date Format Note

The module uses dayfirst=True (international format: DD/MM/YYYY). If your dates are in US format (MM/DD/YYYY), ambiguous dates like 03/04/2025 will be interpreted as April 3rd, not March 4th. Verify a few rows after processing.

Column Translation

Transformation Module

src/transformation/translate_columns.py

Purpose

Translates text content in selected columns from one language to another using Google Translate. Designed for international media analytics where coverage arrives in multiple languages and must be unified for analysis.

Key Features

  • Auto-language detection — Detects source language per cell; skips text already in target language.
  • Similarity grouping — Groups similar texts (85%+ TF-IDF cosine similarity) and translates only the representative, applying the result to all group members.
  • Translation caching — Identical strings are translated once and cached for the session.
  • Parallel processing — Up to 10 concurrent threads for faster execution.
  • Original preservation — Translations are placed in new columns prefixed with T_, keeping originals intact.

CSV Handler Utility

Utility Module

src/utils/csv_handler.py

Purpose

The foundational I/O layer for the entire application. Every module uses the CSV Handler to read and write data files. It handles the messy reality of real-world CSV files — different encodings, delimiters, and formats — so other modules don't have to.

Functions

read_csv(file_path)

Reads a CSV or Excel file with intelligent fallback:

  1. Tries encodings: utf-8 → utf-8-sig → latin1 → cp1252
  2. For each encoding, tries delimiters: tab → comma → semicolon
  3. If all CSV attempts fail, tries reading as Excel (.xlsx / .xls)

write_csv(data, file_path)

Writes a pandas DataFrame to CSV (UTF-8, no index column).

append_csv(data, file_path)

Appends rows to an existing CSV without writing a header. Useful for incremental data collection.

read_csv_to_dict(file_path)

Returns data as a list of dictionaries (one per row). Used internally by the Date Metadata module.

Topic Clustering

Analysis Module

src/clustering/topic_clustering.py

Purpose

Automatically groups text data (headlines, summaries, descriptions) into thematic clusters using machine learning. It applies TF-IDF vectorization and K-Means clustering to discover hidden topics within your media dataset, then provides rich visualizations to help you understand and present the results.

Key Features

  • Text Cleaning — Optional removal of HTML tags, URLs, email addresses, and punctuation before analysis.
  • TF-IDF Vectorization — Converts text into numerical vectors that capture the importance of each word relative to the full corpus.
  • K-Means Clustering — Groups documents into a configurable number of topic clusters.
  • Auto-Detection — Automatically identifies the text column if not specified.
  • Rich Visualizations — Cluster distribution, PCA scatter plots, top terms per cluster, heatmaps, pie charts, and model metrics.

How to Interpret Results

Labeling Clusters

Clusters are assigned numeric IDs (0, 1, 2, ...). To give them meaningful names, check the Top Terms for each cluster — these are the most distinctive keywords. Read a few sample articles from each cluster to confirm the theme, then rename in your analysis (e.g., Cluster 0 = "Economy", Cluster 1 = "Politics").

Choosing Cluster Count

Start with 5 clusters (default). If topics feel too broad, increase. If they overlap, decrease. Check the inertia score — lower is better, but with diminishing returns.

No internet required — Unlike translation, clustering runs entirely on your local machine using scikit-learn.

CSV Merge

Merge Module

src/merge/merge_csv.py

Purpose

Combines multiple CSV files into a single consolidated dataset. Each file is tagged with a Country Code derived from the first two characters of its filename, making it ideal for consolidating international media coverage data.

When to Use

  • You have separate media coverage files per country (e.g., PH_coverage.csv, US_coverage.csv)
  • You need a unified dataset for cross-country analysis
  • You want automatic source identification via the Country Code column

Quick Start Guide

Get up and running in minutes. This guide walks you through installation, launching the app, and processing your first dataset.

Step 1: Install Dependencies

1

Open a terminal and navigate to the project folder:

cd sevengen-internshipweek4.2
pip install -r requirements.txt

Step 2: Launch the Application

2

Run the app using the main entry point:

python main.py

Or launch Streamlit directly:

streamlit run app.py

Your browser will open to the Streamlit dashboard.

Step 3: Upload Your File

3

Use the file uploader in the Streamlit interface to upload your CSV or Excel file. The app accepts:

  • CSV files (any encoding: UTF-8, Latin-1, etc.)
  • Excel files (.xlsx)
  • Tab-separated or semicolon-separated files

Step 4: Select Pipeline Steps

4

Choose which processing steps to apply:

5

Click "Run Pipeline". Watch the progress bar as each step executes. When complete:

  • Preview the first 100 rows in the browser
  • Download as CSV or Excel using the download buttons

Recommended Pipeline Order for Media Data

Best Practice Pipeline
  1. Merge CSV Files — If you have multiple files, merge them first to create a unified dataset.
  2. Normalize Headers — Ensures consistent column names for all subsequent steps.
  3. Remove Duplicates — Eliminates redundant entries before enrichment.
  4. Add Date Metadata — Enriches your data with time components for trend analysis.
  5. Translate Columns — Translates only the unique, cleaned data.
  6. Topic Clustering — Discovers themes in the final, clean dataset (most meaningful when run last).

Adding New Modules

The application is designed for easy extensibility. Follow this guide to add a new processing module to the pipeline.

Step 1: Choose a Category

Decide where your module belongs:

CategoryDirectoryPurpose
Cleaningsrc/cleaning/Standardizes or removes data without adding new information
Transformationsrc/transformation/Enriches or converts data (adds columns, translates, etc.)
Clusteringsrc/clustering/Groups or categorizes data (future)
Mergesrc/merge/Combines multiple datasets (future)
Utilitysrc/utils/Shared helper functions (not a pipeline step)

Step 2: Create the Module File

Create a new Python file in the appropriate directory. Follow this template:

# src/cleaning/my_new_module.py  (or src/transformation/, etc.)

from src.utils.csv_handler import read_csv

def my_new_function(file_path=None, **kwargs):
    """
    Brief description of what this module does.

    Parameters:
        file_path (str): Path to the CSV or Excel file.
        **kwargs: Any additional parameters your module needs.

    Returns:
        pd.DataFrame: The processed DataFrame.
    """
    # 1. Read the data
    if file_path is None:
        from config import RAW_DATA_FILE
        file_path = RAW_DATA_FILE

    df = read_csv(file_path)

    # 2. Apply your transformation
    # ... your processing logic here ...

    # 3. Return the result
    return df

Step 3: Register in __init__.py

Add your function to the appropriate __init__.py for easy importing:

# src/cleaning/__init__.py  (add your import)
from .my_new_module import my_new_function

Step 4: Add to the Pipeline in app.py

Add your module as a new pipeline step in the Streamlit app. Look for the section where steps are defined and add yours following the existing pattern:

# In app.py, add to the pipeline steps:
# 1. Add a checkbox for the user
run_my_step = st.checkbox("My New Step")

# 2. Add the execution logic in the pipeline loop
if run_my_step:
    from src.cleaning.my_new_module import my_new_function
    df = my_new_function(file_path=temp_file)
    write_csv(df, temp_file)

Architecture Conventions

  • Every module function should accept file_path as its first parameter (with a default fallback to config).
  • Every module function should return a pd.DataFrame.
  • Use csv_handler.read_csv() for reading — never pd.read_csv() directly.
  • Keep modules independent — they should not import or depend on other pipeline modules.
  • Handle errors gracefully — raise clear exceptions with user-friendly messages.
